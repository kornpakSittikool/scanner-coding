# RepoScanner VS Code Extension 🗺️

**RepoScanner** คือเครื่องมือสร้างสารบัญและดักจับโครงสร้างโค้ดแบบเพิ่มหน่วย (Incremental AST Codebase Scanner) ซึ่งจะทำหน้าที่สร้างแผนที่และสารบัญย่อยเพื่อลดการใช้งาน Token ของ AI Agent ได้สูงสุดถึง **90% - 99%**

---

## 🛠️ ขั้นตอนหลังการ Clone โปรเจกต์เพื่อนำไปใช้งาน

หากคุณหรือนักพัฒนาคนอื่นทำการ Clone โฟลเดอร์โครงการนี้ไป ให้ทำตามขั้นตอนด้านล่างนี้เพื่อตั้งค่า คอมไพล์ และติดตั้งส่วนต่อขยาย:

### 1. ติดตั้ง Dependencies
เปิด Terminal ในโฟลเดอร์โครงการที่โคลนมา แล้วติดตั้งมอดูลทั้งหมด:
```bash
npm install
```

### 2. คอมไพล์โค้ด (Compile)
ทำการแปลงโค้ด TypeScript (.ts) ของส่วนต่อขยายไปเป็น JavaScript (.js) สำหรับรันใน VS Code:
```bash
npm run compile
```

### 3. พัฒนาและทดสอบสดใน VS Code (Development Mode)
หากต้องการทดสอบโค้ดหรือแก้ไขเพิ่มเติมขณะพัฒนา:
1. เปิดโฟลเดอร์โครงการนี้ด้วย VS Code
2. กดปุ่ม `F5` บนคีย์บอร์ด (หรือไปที่เมนู **Run & Debug** เลือก **Launch Extension**)
3. VS Code จะเปิดหน้าต่างใหม่ขึ้นมา (Extension Development Host) ซึ่งจะมี RepoScanner รันทำงานอยู่ด้านในเพื่อทดสอบสแกนโค้ดและแก้ไขได้ทันที

### 4. แพ็กเกจเป็นไฟล์ส่วนขยาย (Build VSIX)
หากต้องการสร้างไฟล์ติดตั้งจริง `.vsix` เพื่อนำไปใช้งานถาวรบนเครื่อง:
```bash
npm run package
```
*จะได้ไฟล์ผลลัพธ์สำเร็จรูปเป็น `repo-scanner-1.0.0.vsix` ในรูทของโครงการ*

### 5. วิธีการติดตั้งส่วนต่อขยายใน VS Code
คุณสามารถติดตั้งไฟล์ VSIX ที่เจเนอเรตขึ้นมาเข้าไปใน VS Code ด้วยวิธีใดวิธีหนึ่ง:
* **ผ่าน Command Line (สะดวกที่สุด)**:
  ```bash
  code --install-extension repo-scanner-1.0.0.vsix
  ```
* **ผ่านหน้าจอ VS Code (Manual)**:
  1. เปิด VS Code ขึ้นมา ไปที่แถบ **Extensions** (หรือกด `Ctrl+Shift+X`)
  2. คลิกที่ปุ่มจุดสามจุด `...` ที่มุมบนขวาของช่อง Extensions
  3. เลือกเมนู **"Install from VSIX..."**
  4. เลือกไฟล์ `repo-scanner-1.0.0.vsix` เพื่อติดตั้ง

---

## ⚙️ วิธีการใช้งานในโปรเจกต์งานของคุณ

1. ทันทีที่ติดตั้งเสร็จ ให้ทำการ **Reload Window** หรือเปิดโฟลเดอร์โปรเจกต์งานใหม่ขึ้นมา
2. สังเกตแถบซ้ายมือ (Activity Bar) จะมีไอคอน **RepoScanner Control Panel** ปรากฏขึ้น
3. ระบบจะทำงานและสแกนโครงสร้างและตรวจเช็กคลาส/ความสัมพันธ์ของเฟรมเวิร์กในเบื้องหลังทันที
4. **ความคุ้มค่าด้าน Token**: ระบบจะสร้างและอัปเดตไฟล์ `AI_BLUEPRINT.md` (ลายแทงนำทางระบบสำหรับ AI) ใน root โปรเจกต์ให้แบบเรียลไทม์เมื่อบันทึกไฟล์ ช่วยลดขนาด Token ที่ AI ต้องอ่านโครงสร้างระบบลงได้อย่างมหาศาล!

### 🚫 การปรับแต่งการข้ามสแกน (Custom Exclusions)
ระบบจะทำการตรวจจับแคชของ Angular (`.angular`), Next.js (`.next`), Nuxt (`.nuxt`) และอื่น ๆ ให้อัตโนมัติ หากนายต้องการยกเว้นโฟลเดอร์เพิ่มเติมด้วยตนเอง:
1. เปิดไฟล์ `.repo-scanner/config.json` ในโปรเจกต์งานนั้น ๆ
2. เพิ่มโฟลเดอร์ที่ต้องการละเว้นลงในลิสต์ เช่น:
   ```json
   {
     "port": 9876,
     "exclude": [
       "node_modules",
       "dist",
       ".git",
       ".repo-scanner",
       "public/assets/images"
     ]
   }
   ```
